## 현재 기준

- 메인 공개 경험은 `쇼룸형 홈페이지`입니다.
- `쇼룸형 홈페이지`는 정의를 유지하지만, 현재 구현 작업은 당분간 보류합니다.
- `내부쇼룸`은 `/showroom`, `오픈쇼룸`은 `/public/showroom`으로 구분합니다.
- 로컬에서 `내부쇼룸`을 수정할 때는 로그인 대신 현재 우회 진입 경로를 사용합니다.
- 토큰 기반 외부 쇼룸은 메인 채널이 아니라 보조 채널입니다.
- 채널톡은 링크 재전송보다 `관심 사례 기반 후속 정보`를 제공하는 역할로 재정의했습니다.
- 공개 동선과 내부 운영 동선은 반드시 분리합니다.
- 이미지 자산 관리는 쇼룸, 콘텐츠 자동화, 상담 보조의 공통 허브로 간주합니다.

## 최근 결정 로그

### 2026-04-06 — 쇼룸 경로 정의 및 로컬 수정 규칙 정리

- `쇼룸형 홈페이지`는 고객 공개 메인 경험 정의를 유지하되, 당분간 구현 작업은 보류하기로 했습니다.
- `내부쇼룸`은 `/showroom`, `오픈쇼룸`은 `/public/showroom`으로 명확히 구분합니다.
- 현재 로컬 로그인 흐름이 작업 중인 로컬 화면 대신 배포된 Vercel 주소로 이동시키는 문제가 있어, `내부쇼룸` 수정은 로그인 대신 우회 진입 경로 기준으로 진행합니다.
- 이 규칙은 로컬에서 수정하고 커밋/푸시로 배포본을 고쳐야 하는 현재 작업 방식에 맞춘 임시 운영 기준입니다.

### 2026-03-27 — 쇼룸형 홈페이지 메인화 방향 확정

- 고객 공개 동선의 중심을 `쇼룸형 홈페이지`로 재정의했습니다.
- 기존 `외부 발송용 쇼룸`은 상담 중 특정 사례를 다시 좁혀 보여주는 보조 수단으로 내렸습니다.
- 채널톡은 링크 발송보다 `추가 정보 제공 + 행동 촉구`를 담당하는 후속 정보 레이어로 정리했습니다.
- 후속 정보의 최소 단위는 `좌석 수`, `평수`, `예산 밴드`, `페인포인트`, `해결 포인트`, `운영자 후기`입니다.
- 최소 행동 이벤트 초안은 `showroom_open`, `showroom_view_case`, `showroom_view_before_after`, `showroom_reply_intent`입니다.

### 2026-03-15 — 채널톡 운영 전략 재정의

- 채널톡 1차 도입은 AI 대화형이 아니라 `폼 기반 질문 수집`으로 확정했습니다.
- 질문 순서는 `도움 유형 -> 업종 -> 면적 -> 지역 -> 전화번호`이며, 전화번호 입력이 완료 게이트입니다.
- 쇼룸은 폼 중간이 아니라 전화번호 입력 후 `기다리시는 동안 보는 대기/설득 페이지`로 사용합니다.
- 채널톡 인입은 기존 `consultations`와 바로 섞지 않고, 장기적으로 별도 인입 대시보드 또는 임시카드로 분리합니다.
- 전체 OS는 백오피스 완성 전에도 공개 동선만 먼저 단계적으로 퍼블리시할 수 있습니다.

### 2026-03-12 — 쇼룸 정교화

- 쇼룸은 `/showroom` 단일 기준의 탐색형 랜딩 페이지로 유지합니다.
- 공감카드와 검색창의 역할을 분리했습니다.
- `Before/After` 자산은 일반 카드 흐름에 섞지 않고 전용 섹션으로 격리합니다.
- 쇼룸 문의는 `showroom_context`, `showroom_entry_label`을 남겨 담당자가 유입 문맥을 즉시 파악할 수 있어야 합니다.

### 2026-03-12 — Google Chat 견적 자동화 안전 롤백

- Google Chat 견적 자동화는 구현 자산으로는 남기되, 현재 운영 기본 흐름에서는 제외했습니다.
- 라이브 `n8n`과 로컬 `gas/n8n-workflow.json`은 모두 롤백 상태로 맞췄습니다.
- 현재 운영 기준은 상담카드/견적 관리 + Takeout 기반 수동 보조 흐름입니다.

### 2026-03-11 — 상담 관리 작업형 대시보드 정리

- 상담 관리는 다른 화면과 분리된 `작업형 대시보드`로 봅니다.
- 데스크톱에서는 우측 상세 패널을 기준판처럼 유지합니다.
- 사용자가 직접 바꾸지 않았다면 시스템이 임의로 첫 카드로 돌아가지 않습니다.
- 상태 변경 후 카드가 다른 탭으로 이동해도, 해당 카드가 포함된 페이지를 자동 추적해야 합니다.
- 목록은 페이지당 40개, 상단 제어는 `인입일`, `업데이트일`, `정렬 토글` 3종을 기본으로 봅니다.

### 2026-03-09 — 상담 상태 7종 확정

- 상담 상태는 `접수 / 견적 / 진행 / 완료 / AS / 무효 / 거절` 7종으로 고정합니다.
- `휴식기`는 `완료`로 흡수하고, 추후 필요하면 별도 플래그로 분리합니다.
- 핀 기능은 별도 컬럼이 아니라 `metadata.pinned` 패턴으로 유지합니다.
