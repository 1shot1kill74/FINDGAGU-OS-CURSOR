# 콘텐츠 준비도 가이드

## 1. 목적
- 이 문서는 발행 큐와 콘텐츠 상세에서 보이는 `콘텐츠 준비도`, `자동화 준비도`, `연동 보완` 신호를 해석하는 기준이다.

## 2. 콘텐츠 준비도
- 원문 제목이 있다.
- 핵심 메시지와 페인포인트가 정리되어 있다.
- 공개 수준이 정해져 있다.
- CTA 또는 다음 행동 유도가 있다.
- 최소 1개 이상 연결 자산 또는 사례 근거가 있다.

## 3. 자동화 준비도
- 채널별 배포 초안이 있다.
- 필요한 메타 필드가 비어 있지 않다.
- 선택한 채널의 작업유형이 정해져 있다.
- 웹훅 준비 상태가 최소 `mock 연결` 이상이다.

## 4. 발행 큐에서 보는 법
- `준비 보완`: 원문 또는 메타가 비어 있다.
- `자동화 확인`: 자동화 요청을 만들기 전에 payload와 채널 조건을 다시 봐야 한다.
- `템플릿 보완`: 템플릿 연결이 약하거나 아직 선택되지 않았다.
- `연동 보완`: 환경 설정 때문에 실제 요청 생성이 막혀 있다.
- `마지막 목록 갱신`: 현재 보고 있는 목록 기준 시각이다.
- `연동 상태 확인`: 웹훅 판정 기준 시각이다.
- `새로고침 권장 힌트`: 두 시각 차이를 보고 어떤 버튼을 눌러야 하는지 알려준다.

## 5. 콘텐츠 상세에서 보는 법
- `콘텐츠 준비도`: 원문/기획 측면 준비 수준이다.
- `자동화 준비도`: 외부 자동화로 넘길 수 있는 상태인지 보여준다.
- `준비 체크리스트`: 지금 비어 있는 핵심 항목을 직접 알려준다.
- `추천 액션`: 실패 작업, 배포 오류, 후킹/FAQ 부족, 미생성 채널 같은 신호를 바탕으로 지금 먼저 해야 할 일을 알려준다.
- `콘텐츠 진단`: 상태 불일치, 발행 URL 누락, 실패 작업, 배포 오류, 파생 초안 누락 같은 이상 신호를 먼저 보여준다.
- `웹훅 준비`: 현재 채널 기준 실제 요청 가능 상태다.
- `마지막 콘텐츠 갱신`: 본문과 메타 정보 기준 시각이다.
- `연동 상태 확인`: 웹훅 판정 기준 시각이다.
- `마지막 작업 목록 갱신`: 최근 자동화 작업 패널 기준 시각이다.
- `새로고침 권장 힌트`: 세 시각 차이를 보고 필요한 새로고침 버튼을 안내한다.

## 6. 큐 카드 해석
- `실패 작업 n건`: 자동화 큐를 먼저 확인해야 하는 콘텐츠다.
- `배포 오류 n건`: 배포 관리에서 채널 상태와 URL 반영을 다시 봐야 한다.
- `발행 완료 n건`: 이미 외부 채널 반영이 시작된 콘텐츠다.
- 위 신호는 우선 분류 배지와 함께 읽어야 하며, 하나의 콘텐츠가 여러 신호를 동시에 가질 수 있다.

## 7. 운영 원칙
- `연동 미설정`이면 입력이 충분해도 요청 생성은 막는 것이 맞다.
- `mock 연결`이면 내부 검증은 가능하지만 운영 완료로 보지 않는다.
- `실URL 연결`이어도 payload가 빈약하면 자동화 준비 완료로 판단하지 않는다.
- `발행 URL 있음`과 `웹훅 준비됨`은 같은 뜻이 아니다. 이미 발행된 URL이 있어도 자동화 webhook 준비 상태는 별도로 확인해야 한다.
- 준비도 점수는 자동 발행 허가가 아니라, 운영자 우선순위 판단을 돕는 신호로 사용한다.
