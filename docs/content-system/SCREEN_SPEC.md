# 콘텐츠 시스템 화면 명세

## 1. 목적
- 이 문서는 콘텐츠 시스템의 핵심 5개 운영 화면에 대한 최소 구현 기준을 정리한다.

## 2. 공통 원칙
- 각 화면은 새로고침 없이도 부분 재조회가 가능해야 한다.
- `마지막 ... 갱신` 문구를 통해 지금 보고 있는 데이터 기준 시각을 보여줘야 한다.
- 연동 판정이 따로 있는 화면은 `연동 상태 확인` 또는 `확인 시각`을 별도로 보여줘야 한다.
- 두 시각이 크게 벌어지면 화면은 실제 시각 비교 결과를 바탕으로 어떤 버튼을 다시 눌러야 하는지 한 줄 힌트로 알려줘야 한다.
- 운영 패널의 추천 후보, 막힘 신호, 관련 액션은 가능한 한 해당 화면/탭/채널로 바로 이동하는 딥링크를 제공해야 한다.
- 주요 화면은 `운영 진단` 또는 `자동 진단 요약`을 통해 상태 불일치, URL 누락, 실패 사유 누락 같은 이상 신호를 먼저 보여줄 수 있어야 한다.

## 3. 발행 큐
### 목적
- 오늘 처리할 콘텐츠를 고르고, 우선순위와 차단 신호를 빠르게 분류한다.

### 필수 영역
- KPI 카드
- 검색/상태/업종/우선 분류 필터
- `쇼룸에서 동기화`
- `오늘 할 일 3줄`
- `공유용 요약`
- `실운영 최종 점검`
- `데이터 원천 표시`
- `목록 새로고침`
- `연동 상태 새로고침`
- `마지막 목록 갱신`
- `연동 상태 확인`
- `새로고침 권장 힌트`
- 좌측 콘텐츠 목록과 우측 선택 항목 운영 패널 구조
- 우측 패널의 `우선 처리 근거`, `준비 체크`, `막힘 신호`, `최근 활동`

### 동작 기준
- `쇼룸에서 동기화`를 누르면 내부 쇼룸 원천 이미지 기준으로 현장별 콘텐츠 후보를 가져와야 한다.
- 동기화 후 신규 콘텐츠는 발행 큐 상단부에 즉시 나타나야 한다.
- 동기화로 생성된 신규 콘텐츠는 기본 배포 채널 뼈대가 함께 준비되어 상세/배포 화면에서 빈 상태로 보이지 않아야 한다.
- 동기화로 생성된 신규 콘텐츠는 기본 파생초안도 함께 준비되어 상세 화면의 `파생 콘텐츠` 탭에서 바로 보일 수 있어야 한다.
- `오늘 할 일 3줄`은 고정문구가 아니라 현재 필터 결과와 우선 분류 기준으로 자동 생성되어야 한다.
- 화면은 현재 기준이 `로컬 워크스페이스`인지 `Supabase 원천 연결`인지 한 줄로 보여줘야 한다.
- 목록 카드에서는 최소 `다음 액션`, `채널 수`, `파생 초안 수`, `실패/오류/발행 완료 배지`가 보여야 한다.
- `실운영 최종 점검`은 최소 `critical 진단 없음`, `실발행 URL 확보`, `자동화 완료 흐름 확인`, `운영 이력 추적 가능`, `잔여 실패 작업 정리` 기준을 한 번에 보여줘야 한다.

## 4. 콘텐츠 상세
### 목적
- 하나의 콘텐츠를 기획, 원문 작성, 파생 제작, 배포 준비까지 끝낸다.

### 필수 탭
- 기본 정보
- 블로그 원문
- 파생 콘텐츠
- 배포 상태

### 필수 보조 정보
- `콘텐츠 준비도`
- `자동화 준비도`
- `웹훅 준비`
- `마지막 콘텐츠 갱신`
- `연동 상태 확인`
- `마지막 작업 목록 갱신`
- `새로고침 권장 힌트`
- `파생 콘텐츠 탭`에서는 파생 초안 목록과 채널/상태/후킹/개요를 볼 수 있어야 한다.
- `배포 상태 탭`에서는 채널 선택 칩, 채널 목록, 우측 상세 패널 구조가 있어야 한다.
- 우측 상세 패널에는 `발행 준비 체크`, `채널 파생 초안`, `최근 자동화 작업`, `최근 활동 로그`가 함께 보여야 한다.
- `배포 상태 탭`의 상태 배지는 색과 텍스트 모두로 의미가 분명해야 한다.
- 주요 저장/재요청/상태 변경 조작은 활동 로그에 자동 기록되어야 한다.

## 5. 배포 관리
### 목적
- 채널별 배포 상태와 발행 URL을 한 화면에서 관리한다.

### 필수 요소
- 채널 필터
- 상태 필터
- `목록 새로고침`
- `연동 상태 새로고침`
- `마지막 목록 갱신`
- `연동 미설정 / 생성 불가 / 확인 필요` 신호
- 좌측 채널 목록과 우측 상세 패널 구조
- 우측 상세 패널의 `발행 준비 체크`
- 우측 상세 패널의 `최근 자동화 작업`, `최근 활동`, `채널 파생 초안`
- 우측 상세 패널의 `상태 자동 반영 결과`
- 상태별 의미가 드러나는 배지 색상

## 6. 자동화 큐
### 목적
- 자동화 요청의 상태, payload, 재요청, 디스패치를 한 화면에서 관리한다.

### 필수 영역
- `연동 준비 상태` 배너
- `실운영 준비도` 요약 (`운영 시작 가능 / 부분 준비 / mock 단계 / 미설정`)
- 채널별 준비 카드 (`채널명`, `준비 단계`, `콘텐츠/배포/작업 수`, `실패 수`)
- 작업유형별 준비 카드
- 검색/상태/채널/작업유형 필터
- `새로고침`
- `상태 새로고침`
- `실패 항목 일괄 재요청`
- `확인 시각`
- `마지막 목록 갱신`
- `새로고침 권장 힌트`
- 우측 상세 패널의 payload 편집 영역
- 우측 상세 패널의 `상태 자동 반영 결과`

## 7. 템플릿
### 목적
- 블로그, CTA, 카드뉴스, 쇼츠, 롱폼 템플릿을 관리하고 재사용한다.

### 필수 요소
- 템플릿 목록
- 템플릿 편집기
- 성능/사용 요약
- 운영 가이드 요약
- 운영 체크
- 추천 적용 후보 콘텐츠 목록
- `데이터 새로고침`
- `마지막 데이터 갱신`

## 8. 비기능 기준
- 버튼을 눌렀을 때 전체 화면이 과하게 번쩍이지 않아야 한다.
- 오래된 응답이 최신 상태를 덮어쓰지 않도록 재조회 가드가 필요하다.
- 상태 배지는 색만이 아니라 텍스트로도 의미가 분명해야 한다.
