# 자동화 운영 런북

## 1. 먼저 보는 순서
- `자동화 큐 > 연동 준비 상태`
- `발행 큐 > 연동 보완`
- `콘텐츠 상세 > 웹훅 준비`
- `배포 관리 > 연동 미설정 / 생성 불가`

## 2. 자주 쓰는 버튼
- `상태 새로고침`: 자동화 큐의 웹훅 준비 상태를 최신 스냅샷 기준으로 다시 계산한다.
- `새로고침`: 자동화 큐 목록을 다시 불러온다.
- `연동 상태 새로고침`: 발행 큐, 콘텐츠 상세, 배포 관리에서 현재 웹훅 판정을 최신 스냅샷 기준으로 다시 본다.
- `목록 새로고침 / 콘텐츠 새로고침 / 작업 새로고침`: 실제 데이터 재조회용이며, 연동 상태 판단도 함께 최신 기준으로 맞춰진다.
- `웹훅 호출`: 현재 queued 요청을 즉시 디스패치한다.
- `재요청`: 같은 조건으로 새 큐 항목을 다시 적재한다.
- `실패 항목 일괄 재요청`: 현재 자동화 큐 필터 범위 안의 failed 작업을 한 번에 다시 적재한다.

## 3. 준비 상태 판정 해석
- `운영 시작 가능`: 실URL과 secret이 모두 갖춰진 상태다.
- `부분 준비`: 일부 작업유형 또는 일부 채널만 준비된 상태다.
- `mock 단계`: 테스트 URL 또는 mock 흐름으로만 연결된 상태다.
- `미설정`: URL 또는 secret이 빠져 실제 요청 생성이 불가능한 상태다.
- 이미 발행 URL이 들어온 채널이라도 webhook 준비 상태는 별도 기준으로 봐야 한다.

## 4. 화면별 해석
### 발행 큐
- `연동 보완`: 지금 단계에서 필요한 자동화 타입이 아직 준비되지 않았다.
- `마지막 목록 갱신`: 현재 보고 있는 큐 목록이 언제 기준인지 보여준다.
- `연동 상태 확인`: 현재 웹훅 판정이 언제 기준인지 보여준다.
- `새로고침 권장 힌트`: `목록`과 `연동 상태` 중 무엇을 먼저 갱신해야 하는지 알려준다.

### 콘텐츠 상세
- `웹훅 준비`: 선택 채널 기준 실제 요청 가능 상태다.
- `마지막 콘텐츠 갱신`: 본문과 메타 정보 기준 시각이다.
- `연동 상태 확인`: 웹훅 판정 기준 시각이다.
- `마지막 작업 목록 갱신`: 최근 자동화 작업 패널 기준 시각이다.
- `새로고침 권장 힌트`: 세 시각 차이를 보고 어떤 버튼을 눌러야 하는지 알려준다.
- `콘텐츠 진단`: 상태 불일치, 발행 URL 누락, 자동화 실패, 배포 오류, 파생 초안 누락 신호를 먼저 보여준다.

### 자동화 큐
- `확인 시각`: 연동 준비 상태를 마지막으로 확인한 시각이다.
- `마지막 목록 갱신`: 현재 큐 목록을 다시 불러온 시각이다.
- `새로고침 권장 힌트`: `상태 새로고침`과 `새로고침` 중 무엇이 필요한지 알려준다.
- `자동 진단 요약`: 현재 워크스페이스 기준 상태 불일치와 누락 신호의 개수를 보여준다.

### 배포 관리
- `연동 미설정`: 채널 상태를 편집하더라도 실제 자동화 생성은 불가능한 상태다.
- `생성 불가`: 현재 선택 채널에서 요청을 만들 수 없는 상태다.
- `마지막 목록 갱신`: 현재 배포 보드의 기준 시각이다.

## 5. 실패 유형별 대응
### 웹훅 URL 미설정
- 환경변수와 secret 값을 먼저 확인한다.
- 수정 직후 `연동 상태 새로고침` 또는 `상태 새로고침`을 반드시 실행한다.
- 관련 화면의 `마지막 ... 갱신` 문구가 현재 시점으로 바뀌는지 함께 확인한다.

### payload 누락
- 콘텐츠 상세의 `자동화 Payload 미리보기`로 돌아가 빠진 필드를 확인한다.
- 템플릿 선택, 채널 메타, 공개 수준, CTA 관련 값이 비어 있지 않은지 본다.

### 장시간 processing
- 외부 자동화가 실제로 처리 중인지 먼저 확인한다.
- 큐 목록만 오래된 것일 수 있으니 `새로고침`을 먼저 실행한다.
- 반영 시각이 오래됐고 외부 시스템도 멈췄다면 재요청 또는 수동 callback 테스트를 검토한다.

### 발행 URL 미반영
- 자동화 큐의 최근 반영 메타와 배포 관리의 URL 상태를 교차 확인한다.
- 배포 관리만 오래된 경우 `목록 새로고침`을 먼저 누른다.
- 콘텐츠 상세가 오래된 경우 `콘텐츠 새로고침`으로 마지막 상태를 다시 맞춘다.

## 6. 실URL 전환 직후 1회전 점검
1. 자동화 큐에서 대상 작업유형이 `실URL 연결`인지 확인한다.
2. 발행 큐에서 `연동 상태 새로고침`, `목록 새로고침`을 각각 1회 실행한다.
3. 콘텐츠 상세에서 `웹훅 준비`와 `자동화 요청 생성` 가능 여부를 확인한다.
4. 요청을 만든 뒤 최근 자동화 작업과 자동화 큐에 동시에 반영되는지 본다.
5. 배포 관리에서 최종 URL이 들어오고 차단 신호가 사라지는지 확인한다.
