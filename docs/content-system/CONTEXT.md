# 콘텐츠 시스템 컨텍스트

## 1. 왜 필요한가
- 기존 블로그 자동화는 `우리가 이 현장을 했다`는 단순 홍보 흐름에 가까웠다.
- 지금은 쇼룸과 이미지 자산이 이미 콘텐츠 허브 역할을 하므로, 더 전략적인 발행 시스템이 필요하다.
- 목표는 블로그 1개를 중심으로 카드뉴스, 쇼츠, 롱폼, SNS 캡션까지 이어지는 멀티채널 운영 체계를 만드는 것이다.

## 2. 운영 해석
- 블로그는 메인 스토리다.
- 카드뉴스는 핵심 요약과 클릭 유도 역할이다.
- 쇼츠는 강한 후킹과 블로그 유입 유도 역할이다.
- 롱폼은 블로그보다 더 깊은 해설 또는 재가공된 설명 역할이다.

## 3. 사용자 관점
- 운영자는 콘텐츠를 직접 쓰기보다 `정리`, `선택`, `검수`, `요청 생성`, `상태 확인`을 빠르게 끝내고 싶어 한다.
- 따라서 각 화면은 화려한 편집기보다 운영 판단과 차단 신호 해석이 쉬워야 한다.

## 4. 중요한 판단 기준
- 자동화는 무조건 발행이 아니라 `운영자가 통제 가능한 반자동`이어야 한다.
- 외부 연동 상태가 불확실할 때는 막연한 성공 메시지보다 명시적인 준비 상태와 갱신 시각이 더 중요하다.
- 페이지를 다시 들어오지 않아도 최신 상태를 다시 맞출 수 있어야 실무 피로가 줄어든다.

## 5. 연결 관계
- 이미지 자산은 원천 데이터다.
- 쇼룸은 고객 설득 표면이자 콘텐츠 후보 소스다.
- 콘텐츠 시스템은 이 둘을 받아 발행 가능한 구조화 자산으로 바꾸는 내부 운영 허브다.
