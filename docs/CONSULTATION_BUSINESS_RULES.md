# 상담카드 비즈니스 규칙

## 1. 고객 식별 (Customer Identification)

- **규칙**: `customer_phone`(전화번호)이 같은 데이터가 들어오면 **기존 고객**으로 인식
- **적용**:
  - 전화번호 비교 시 숫자만 추출하여 비교 (010-1234-5678 = 01012345678)
  - 등급 자동 승격, 재구매 판단 등에 사용

---

## 2. 재구매 관리 (Repeat Purchase)

- **규칙**: 같은 번호이지만 **새로운 `project_name`과 `start_date`**가 들어오면 → **새로운 상담 카드** 생성
- **금지**: 기존 카드 덮어쓰기
- **구현**:
  - `project_name`은 UNIQUE 제약 → 동일 프로젝트명 중복 불가
  - 신규 상담 = 항상 INSERT (기존 행 UPDATE로 덮어쓰지 않음)
  - `created_at`: 최초 생성 시각 (시스템 자동)
  - `start_date`: 프로젝트 시작일 (업무상 입력)

---

## 3. 등급 자동 승격 (Grade Auto-Promotion)

- **규칙**: 동일 전화번호로 **성공적으로 완료된** 프로젝트가 **2개 이상**이면 `customer_grade`를 자동으로 **'단골'**로 설정
- **완료 기준**: `status` IN ('시공완료', '휴식기', '계약완료')
- **구현**: DB 트리거
  - `AFTER INSERT`, `AFTER UPDATE OF customer_phone`, `AFTER UPDATE OF status` 시 실행
  - 승격 대상: `customer_grade`가 `NULL`, `''`, `'신규'`인 행만
  - **파트너는 절대 덮어쓰지 않음** (수동 지정 우선)

## 4. 파트너 수동 지정

- **규칙**: 관리자가 상세 페이지에서 '파트너 지정' 버튼 클릭 시 `customer_grade`를 **'파트너'**로 고정
- **우선순위**: 자동 단골 판정보다 우선. 파트너로 지정되면 자동 로직에서 제외
- **시각화**: 파트너 카드는 상황판에서 테두리 색(골드/앰버) + 이름 옆 ⭐ 아이콘

---

## 컬럼 활용 요약

| 컬럼 | 용도 |
|------|------|
| `created_at` | 상담카드 생성 시각 (시스템) |
| `start_date` | 프로젝트 시작일 (업무 입력) |
| `customer_phone` | 고객 식별·재구매·등급 승격 기준 |
| `project_name` | 프로젝트별 고유 식별 (UNIQUE) |
| `customer_grade` | 신규 → 단골 자동 승격 대상 |
