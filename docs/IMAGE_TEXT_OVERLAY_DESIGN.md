# 한글 텍스트 레이어 합성 — 기술 설계서

과거 Cloudinary 한글 텍스트 합성 실패 사례를 반영하여, **가장 안정적이고 유지보수가 쉬운** 방식을 채택한다.

---

## 1. 폰트 종속성 해결 (권장: 클라이언트 합성)

### 문제
- Cloudinary 서버 측 기본 폰트는 한글 지원이 불안정하거나 폰트 목록에 의존한다.
- 서버 폰트에 의존하면 환경 변경 시 한글 깨짐이 재발할 수 있다.

### 권장 방식: 클라이언트 사이드 Canvas 합성 후 단일 이미지 업로드
- **시점**: 사진 업로드 **전**, 브라우저에서 처리.
- **방법**: Browser Canvas API로 업체명·워터마크·날짜 등을 이미지에 그린 뒤, `canvas.toBlob()`으로 단일 이미지로 내보내 그 결과만 Cloudinary/Supabase에 업로드한다.
- **장점**:
  - 브라우저가 로드한 웹 폰트(한글)를 그대로 사용 → **한글 출력이 가장 확실**.
  - 업로드되는 것은 픽셀 이미지뿐이므로 Cloudinary/Storage는 **텍스트 레이어를 전혀 다루지 않음**.
- **구현 시 검토 사항**:
  - Canvas 2D 컨텍스트, `measureText`, 원하는 폰트 로드(`FontFace` 또는 CSS).
  - 해상도 유지를 위해 원본 크기 또는 2x 등 스케일 고려.
  - 추출/내보내기 진입점에서 **텍스트 파라미터(업체명, 날짜 등)**를 받을 수 있도록 인터페이스를 열어둠 → `ImageExportOptions`(또는 동일 역할 타입) 참고.

---

## 2. Named Transformation 활용 (텍스트 제외)

- Cloudinary **Named Transformation**은 **이미지 가공 전용**으로만 사용한다.
  - 리사이징, 압축, 품질, 포맷 변환, 필터 등.
- **가변적인 한글 텍스트**(업체명, 날짜, 캠페인 문구 등)는 Named Transformation으로 합성하지 않는다.
- 텍스트는 **DB/메타데이터와 연동**하여:
  - 위 1안처럼 **클라이언트에서 합성 후 업로드**, 또는
  - 아래 3안처럼 URL 오버레이를 쓸 경우 **인코딩·폰트 시나리오**를 엄격히 따른다.

---

## 3. 대안: Cloudinary URL 기반 텍스트 합성 시

나중에 Cloudinary URL에서 텍스트 오버레이를 시도할 경우를 위한 설계만 정리한다. **(현재 구현하지 않음)**

### 3.1 한글 인코딩
- URL에 한글 텍스트를 넣을 때 **인코딩** 필수.
- **중첩 `encodeURIComponent`** 사용 시나리오를 고려:
  - Cloudinary가 내부적으로 한 번 더 디코딩하는 경우 등에 대비.
  - 예: `l_text:폰트_스타일:${encodeURIComponent(encodeURIComponent(한글텍스트))}` (실제 구문은 Cloudinary 문서 기준으로 적용).
- 쿼리/경로에 따라 단일 `encodeURIComponent`로 충분할 수 있으므로, 도입 시 Cloudinary 문서와 실제 동작을 반드시 검증할 것.

### 3.2 사용자 폰트 업로드 시나리오
- Cloudinary에 **한글 지원 사용자 정의 폰트**를 업로드하여 사용하는 방식.
- 설계 시 포함할 내용:
  - 폰트 업로드 절차(대시보드 또는 API).
  - `l_text:...` 등에서 해당 폰트명/스타일 지정 방법.
  - 폰트 버전·이름 변경 시 URL/네이밍 정책.
- 이 방식도 **서버 폰트 의존성**이 생기므로, 한글 안정성 확보가 어렵다면 1안(클라이언트 합성)을 우선한다.

---

## 4. 이미지 추출 도구와의 연동

- **이미지 추출/내보내기** 기능 설계 시, **텍스트 합성이 나중에 추가될 수 있음**을 전제로 한다.
- 따라서 **텍스트 데이터(업체명, 날짜, 워터마크 문구 등)**를 파라미터로 넘길 수 있는 **인터페이스만** 열어둔다.
- 코드 상 계약: `ImageExportOptions`(또는 동일 역할 타입)에 선택 필드로 `textOverlay` 등 정의 → 추후 클라이언트 Canvas 합성 또는 Cloudinary URL 합성 시 이 옵션을 사용한다.
- **지금 당장 구현하지 않고**, 호출부와 파라미터 형태만 확정한다.

## 요약

| 구분 | 내용 |
|------|------|
| **권장** | 업로드 전 클라이언트(Canvas)에서 한글 텍스트 합성 → 단일 이미지 업로드 |
| **Named Transformation** | 리사이즈·압축·필터 등 이미지 가공만, 텍스트 제외 |
| **Cloudinary URL 합성 (대안)** | 한글은 `encodeURIComponent`(및 필요 시 중첩), 사용자 폰트 업로드 시나리오를 설계에 포함 |
| **추출 도구** | 텍스트(업체명, 날짜 등)를 넘길 수 있는 인터페이스만 확정, 구현은 추후 |
