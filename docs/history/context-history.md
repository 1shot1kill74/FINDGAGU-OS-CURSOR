# 컨텍스트 아카이브

> 루트 `CONTEXT.md`에서 분리한 과거 컨텍스트 메모입니다.  
> 이 문서는 오래된 기술 스택 설명, 진행 상황 요약, 세부 운영 규칙의 배경을 남겨 둡니다.

## 1. 과거 운영 방향 메모

- 한동안 프로젝트의 직접적인 최우선 목표는 `상담카드/견적 관리 중심 흐름 안정화`였습니다.
- Google Chat 견적 자동 응답은 실험 후 보류 상태로 전환되었습니다.
- 2026-03-15 시점에는 `이미지 자산관리 마이그레이션`이 쇼룸, 블로그 자동화, 유튜브 자동화, 카카오/채널톡 상담 자료 전송의 공통 허브라는 점이 강하게 강조되었습니다.
- 2026-03-27 시점에는 고객 공개 동선의 중심을 `쇼룸형 홈페이지`로 재정의했습니다.

## 2. 기술 스택의 상세 배경

- 프론트엔드는 `React`, `Vite`, `TypeScript`, `Tailwind CSS` 기반입니다.
- 백엔드는 `Supabase`를 사용하며, Auth, Database, Storage, Edge Functions를 함께 사용합니다.
- AI 파싱은 운영 시점에 `Gemini`를 메인, `GPT-4o`를 폴백으로 두는 방향을 검토/적용해 왔습니다.
- 자동화 엔진은 `Make`에서 `n8n`으로 전환되었습니다.
- 개발 방식은 `Cursor` 시각 편집과 `Claude Code` 터미널 에이전트를 병행하는 하이브리드 전제를 유지해 왔습니다.

## 3. 세부 운영 규칙 중 분리된 것

아래 내용은 루트 `CONTEXT.md`에서는 제외하고, 필요 시 이 아카이브나 `BLUEPRINT.md`를 참고합니다.

- 이미지 파일명 규칙과 `base_alt_text` 동기화 방식
- Cloudinary Named Transformation 관련 상세 정책
- `update_date` 해석과 Google Chat lastActiveTime 보조 경로
- Takeout 기반 견적 이미지 브라우저의 세부 운영 해석
- 쇼룸 운영 규칙의 세부 예외 처리
- 채널톡 폼 질문지의 구체적 선택지
- 이미지 자산 관리 화면의 세부 필터/레이어 운영 규칙
