# 설계도 아카이브

> 루트 `BLUEPRINT.md`에서 분리한 과거 상세 설계 메모입니다.  
> 현재 구현 기준은 루트 `BLUEPRINT.md`를 우선하고, 이 문서는 장문 필드 정의와 이전 세부 규칙을 보존하는 용도로만 사용합니다.

## 1. 과거에 루트 설계도에 함께 들어 있던 내용

아래 항목들은 압축 전 루트 설계도에 장문으로 함께 들어 있었습니다.

- 임무 재정의 히스토리
  - GAS AutoAddBot + n8n 자동화 완료 경위
  - 이미지 자산 허브 우선 전략
  - 쇼룸형 홈페이지 메인화
  - Google Chat 견적 자동화 실험과 롤백
- 필드 단위 DB 스키마 설명
  - 사용자/RBAC 세트
  - 상담/리드 관리 세트
  - 견적/시공 사례 세트
  - 운영/마케팅/분석 세트
- UI 세부 규칙
  - 상담 리스트 카드 1~4행 규칙
  - 고객 상세 패널과 아카이브 동작
  - 이미지 자산 뷰어의 필터/레이어 세부 규칙
  - 시공 사례 뱅크와 공유 시스템의 동작 상세
- 자동화 세부 구조
  - 30일 골든타임
  - Wake-up
  - Google Chat/GAS/n8n 연동 상세
  - 채널톡 FAQ/웹훅 상세
  - 구글 시트 동기화 상세

## 2. 압축 후 이동 원칙

루트 `BLUEPRINT.md`에서 제거한 이유는 다음과 같습니다.

- 날짜가 붙은 재정의 로그는 `JOURNAL` 또는 history가 더 적합함
- 필드 단위 장문 정의는 현재 설계 판단보다 세부 구현 기록에 가까움
- 화면의 픽셀 수준 운영 규칙은 모두 루트에 둘 경우 읽기 속도를 크게 떨어뜨림
- 이미 현재 기준이 바뀐 항목과 여전히 유효한 항목이 한 문서에 섞여 있었음

## 3. 계속 유효하지만 루트에서 축약한 규칙

### 상담 관리

- 상담 상태는 `접수 / 견적 / 진행 / 완료 / AS / 무효 / 거절` 7종 기준
- 선택 카드 유지, 우측 패널 고정, 상태 변경 후 페이지 추적은 여전히 유효

### 이미지 자산

- 이미지 자산은 공개 쇼룸/상담 보조/콘텐츠 재활용의 공통 허브
- 공개 노출용 사례와 내부 기술 자산을 섞지 않는 원칙은 유지
- 카드 배지 우선순위와 상단 필터 단순화 원칙도 유지

### 쇼룸

- 공감카드와 검색어 분리
- `Before/After` 별도 문맥 처리
- 쇼룸 문의 문맥 저장

### 채널톡

- 초기 도입은 폼 기반 질문 수집
- 쇼룸 링크 재전송보다 관심 사례 기반 후속 정보가 우선

## 4. 세부 설계가 필요할 때 참조할 범위

아래 항목이 다시 필요하면 현재 구현 파일과 함께 이 아카이브를 참고합니다.

- 상담 카드 줄 단위 레이아웃
- 상세 패널/실측/아카이브 진입 규칙
- `uploadEngine`, `image_assets`, `project_images`, `order_documents` 관련 예전 세부 메모
- 공개 공유 링크와 카카오 공유 흐름
- Google Chat, GAS, n8n, 구글 시트 동기화의 상세 배경
