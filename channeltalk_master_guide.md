[Master Guide] 채널톡(Channel Talk) 통합 운영 및 기술 연동 매뉴얼
본 문서는 채널톡의 서비스 개요, 기술적 연동(SDK/API), 그리고 기초 개념을 집대성한 마스터 가이드입니다. 우리 회사 OS 및 내부 시스템과의 유기적인 연결을 위한 베이스 데이터로 활용합니다.

Part 1. 서비스 개요 및 운영 (Support & Marketing)
채널톡은 고객 상담, 마케팅, 팀 메신저가 결합된 올인원 비즈니스 메신저입니다.

1.1 주요 기능
실시간 채팅 상담: 고객 문의 대응 및 담당자 배정, 태그 관리.

서포트봇(Supportbot): 반복 문의 자동화를 위한 시나리오 기반 챗봇.

CRM 마케팅: 유저 세그먼트를 활용한 타겟 메시지(일회성/캠페인) 발송.

팀 메신저: 내부 협업 및 외부 메신저(카카오, 네이버 등) 통합 관리.

1.2 운영 설정
업무 시간 및 테마: 브랜드 이미지에 맞는 UI 커스텀 및 상담 가능 시간 설정.

상담 폼: 상담 시작 전 고객으로부터 필수 정보(이름, 연락처 등) 수집.

Part 2. 기술 연동 및 개발 (Developers)
우리 회사 서비스(웹/앱) 및 백엔드 시스템과 채널톡을 연결하는 기술적 명세입니다.

2.1 SDK 연동 (Frontend)
플랫폼: JavaScript(Web), iOS, Android, React Native 지원.

유저 동기화 (Boot):

memberId를 통해 우리 서비스의 회원 정보와 채널톡 유저를 매칭.

커스텀 프로필 속성을 전달하여 상담 시 고객 데이터를 바로 확인.

보안 (Member Hash): 유저 정보 변조 방지를 위해 서버 사이드에서 HMAC 알고리즘으로 생성한 memberHash 필수 적용.

2.2 Open API & Webhooks (Backend)
인증: API 호출 시 x-access-key 및 x-access-secret 헤더 사용.

주요 API: Users(조회/수정), Messages(알림톡 발송), Groups(채널 관리).

Webhooks: 상담 발생(chat.create), 메시지 수신(message.create) 등의 이벤트를 우리 서버로 실시간 전송.

2.3 확장 기능 (Channel Apps)
WAM (Web Application Module): 채널톡 내부에 우리 서비스의 웹 모달을 삽입하여 결제, 예약 등을 처리.

Commands: / 명령어를 통한 업무 자동화 트리거 구현.

Part 3. 핵심 개념 및 초기 설정 (Key Concepts)
연동을 시작하기 전 반드시 숙지해야 할 기본 요소입니다.

3.1 플러그인 키 (Plugin Key)
SDK 설치 시 채널을 식별하는 고유 문자열입니다.

확인 경로: 채널 설정 > 일반 설정 > 버튼 설치 및 설정 > 채널톡 버튼 설치.

3.2 이벤트 및 데이터 구조
이벤트 트래킹: 유저의 특정 행동(장바구니 담기, 결제 완료 등)을 수집하여 마케팅 자동화의 트리거로 활용.

Canonical URL: 페이지별 데이터 수집의 정확성을 높이기 위한 표준 URL 설정.

Part 4. 시스템 연동 체크리스트 (Internal OS Integration)
AI가 우리 회사 시스템 설계 시 참고해야 할 가이드라인입니다.

회원 연동: 내부 DB의 회원 고유 번호가 채널톡 memberId로 일관되게 전달되는가?

데이터 동기화: 결제 상태나 회원 등급 변경 시 API를 통해 실시간으로 채널톡 유저 정보가 업데이트되는가?

알림 워크플로우: 상담 발생 시 사내 업무 툴(Slack 등)로 웹훅 알림이 정상적으로 연동되었는가?

보안: 모든 유저 부팅 로직에 memberHash 검증이 포함되었는가?
